## Key Concepts I Learned

* **The Security-Engineered Pattern(Microsoft Recommended)**-\[i] Store your secrets in Azure Key Vault. \[ii] Used RBAC \& Managed Identities. \[iii] Rotate and version secrets centrally. \[iv] Monitor anomalous access with defender.



* **Two access model**- \[i] **Control Plane**: Create/Delete Vault, Configure Network \& Firewall, Key Vault Contributor-Owner. \[ii] **Data Plane**: Secret User, Crypto Officer, Certificates Officer, Administrator.



* **Two types of managed Identities**- \[i] **System Assigned**: One Identity, Once Resource. \[ii] **User Assigned**: One Identity, Many Resource.



* **Trusted Certificate Partners**- Global Sign \& Digi Cert.
